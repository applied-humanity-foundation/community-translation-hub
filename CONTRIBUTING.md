# Contributing to Community Translation Hub

Thank you for your interest in contributing! This project makes digital literacy education accessible across languages, and every contribution helps.

## Ways to Contribute

### Translation
The most impactful way to contribute is by translating content:
- Translate existing documents to a new language
- Improve existing translations
- Review translations submitted by others

See [TRANSLATION_GUIDE.md](TRANSLATION_GUIDE.md) for detailed translation guidelines.

### Code
- Fix bugs or improve the translation interface
- Add new features (see open issues)
- Improve accessibility
- Write tests

### Content
- Write new educational articles
- Update existing content with current information
- Improve clarity and readability

### Documentation
- Improve README, guides, and API documentation
- Add examples and tutorials

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/your-username/community-translation-hub.git`
3. Install dependencies: `npm install`
4. Start development server: `npm run dev`
5. Create a branch: `git checkout -b my-feature`
6. Make your changes
7. Run tests: `npm test`
8. Submit a Pull Request

## Development Setup

- **Node.js** 18 or later
- **npm** 9 or later

```bash
npm install          # Install dependencies
npm run dev          # Start dev server
npm run build        # Production build
npm run lint         # Run linter
npm run typecheck    # Type check
npm test             # Run tests
```

## Code Style

- TypeScript for all source code
- Follow existing patterns in the codebase
- Use meaningful variable and function names
- Keep functions focused and small

## Commit Messages

- Use present tense ("Add feature" not "Added feature")
- Keep the first line under 72 characters
- Reference issue numbers when applicable

## Questions?

Open an issue with the question label, or start a discussion. We're happy to help!
